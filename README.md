# 📊 Projeto de Ciência de Dados – Capítulo 01

## 📌 Descrição:
Este projeto tem como objetivo realizar a **exploração e limpeza de dados** de uma base contendo informações de clientes de um banco taiwanês, com foco na construção futura de um modelo capaz de prever **inadimplência no próximo mês**.

Os dados utilizados incluem informações demográficas, histórico de pagamentos e valores de faturas dos clientes dos meses de abril a setembro

---

## 🎯 Objetivo:
Realizar a Análise e exploração de dados para que posteriormente seja desenvolvido um modelo de Machine Learning que seja capaz de prever se um cliente ficará inadimplente com base em:
- Dados demográficos  
- Histórico de pagamentos  
- Informações financeiras  

---

## 📚 Referências:
- Livro: *Projetos de Ciência de Dados com Python - Sthephen Klosterman*  
- Dataset: https://www.kaggle.com/datasets/uciml/default-of-credit-card-clients-dataset  

---

## 🛠️ Tecnologias Utilizadas
- Python  
- Pandas  
- NumPy  
- Matplotlib  

## 📊 2. Entendimento das Variáveis
Principais colunas:
- `LIMIT_BAL`: Limite de crédito  
- `SEX`: Gênero do cliente
- `EDUCATION`: Nível educacional  
- `MARRIAGE`: Estado civil  
- `AGE`: Idade  
- `PAY_1` a `PAY_6`: Histórico de pagamentos de abril a setembro
- `BILL_AMT1` a `BILL_AMT6`: Valores de fatura  
- `PAY_AMT1` a `PAY_AMT6`: Valores pagos  

---

# 💼 Impacto de Negócio e Aplicações

Este projeto realiza uma análise exploratória, na qual demonstra como os dados podem ser utilizados estrategicamente para **auxiliar em decisões importantes do setor financeiro**. A partir da exploração, limpeza e entendimento das variáveis, é possível extrair insights relevantes que impactam diretamente a gestão de risco e a rentabilidade da empresa.

## 🎯 Principais Aplicações para o Negócio

## 🔍 Previsão de Inadimplência

A análise dos dados permite identificar padrões de comportamento associados ao risco de inadimplência, possibilitando o desenvolvimento de modelos preditivos capazes de antecipar clientes com maior probabilidade de não pagamento.

## 💳 Otimização da Concessão de Crédito

Com base no histórico financeiro e perfil dos clientes, a empresa pode:

* Ajustar limites de crédito de forma mais segura
* Reduzir exposição ao risco
* Melhorar a eficiência na alocação de recursos

## 👥 Análise de Perfil de Clientes

A combinação de dados demográficos e financeiros possibilita a identificação de padrões comportamentais, permitindo uma melhor compreensão dos diferentes perfis de clientes e seus respectivos níveis de risco.

## 📊 Criação de Score de Crédito Interno

A partir do histórico de pagamentos e valores de fatura, é possível desenvolver um sistema de classificação de clientes, segmentando-os em diferentes níveis de risco (baixo, médio e alto).

## 📉 Estratégias de Cobrança Inteligentes

A segmentação dos clientes possibilita a aplicação de estratégias diferenciadas de cobrança, aumentando a eficiência na recuperação de crédito e reduzindo perdas financeiras.

---

## ⚠️ Qualidade dos Dados e Confiabilidade

Durante o projeto, foram identificados pontos críticos relacionados à qualidade dos dados, como:

* Registros duplicados
* Dados faltantes
* Inconsistências em variáveis categóricas

Esses fatores evidenciam a importância de um processo robusto de **tratamento e governança de dados**, essencial para garantir a confiabilidade das análises e modelos preditivos.

A aplicação deste tipo de análise permite à empresa:

* Reduzir prejuízos com inadimplência
* Melhorar a tomada de decisão baseada em dados
* Aumentar a eficiência operacional
* Desenvolver soluções preditivas escaláveis
